# Swing-Stack dashboard

React + Vite + TypeScript client for the Swing-Stack API. Charts are
ECharts plus one hand-built SVG dispersion fan.

## Run

```sh
npm install
npm run dev        # http://localhost:5173, proxies /api to :3000
```

The Rails API must be running on port 3000 with seeded data
(`bin/rails db:prepare db:seed` in the repo root).

## Auth

The app speaks the full JWT flow: login, single-use refresh token
rotation, retry on 401. In dev it signs in as the seeded demo user so
there is no login friction. To exercise the login form, create
`.env.local` with:

```
VITE_AUTO_LOGIN=false
```

See `.env.example` for all variables.

## Testing

End-to-end tests use Playwright. One command boots the whole stack
(Rails with seeded DB, Vite) and runs headless:

```sh
npx playwright test
```

Conventions, written for both people and agents:

- A `setup` project logs in once and shares `storageState`. The backend
  throttles credential posts to 10/min, so tests must not log in per
  test. The two tests in `e2e/auth.spec.ts` that exercise the login form
  are the exception.
- Chart marks and stat values carry kebab-case `data-testid` attributes
  (`fan-dot`, `stat-shots`, `club-row`). Buttons and forms use roles and
  labels; prefer `getByRole` for those.
- Assertions are structural (counts, visibility, attribute state), not
  exact telemetry values, so the suite passes against any seeded export.
- Failures keep a trace and screenshot in `test-results/`. Inspect with
  `npx playwright show-trace <trace.zip>`.

Playwright agent definitions (planner, generator, healer) live in
`.claude/agents/` at the repo root, generated by
`npx playwright init-agents --loop=claude`. Regenerate after Playwright
upgrades.
